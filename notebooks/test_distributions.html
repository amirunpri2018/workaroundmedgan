Evaluation of medGAN Results¶

Data¶

This is using the MIMIC dataset and looking at the ADMISSIONS table versus the DIAGNOSIS table. Each matrix has rows that are patients and columns that are ICD-9 codes. There are two versions that the medGAN system can work with, either a binary version or a count version. In the binary version every value is either 0 or 1 indicating whether the patient has the diagnosis or not. In the count version we can actually see the number of times that diagnosis appears for each patient.

Binary Comparison¶

First, we will look at the binary data version. Below we grab the original binary data and the synthetic binary data and read it in. Then we can look at the shape of each and basic stats about the matrices

Dimension-wise probability performance¶

One of the metrics used in the paper to measure performance is Dimension-wise probability, described as below.

This is a basic sanity check to confirm the model has learned each dimension’s distribution correctly. We use the training set $R$ to train the models, then generate the same number of synthetic samples $S$. Using $R$ and $S$, we compare the Bernoulli success probability $p_k$ of each dimension $k$.

Using this check we can compare the two matrices in the plot below.

In [28]:
orig_col_avg = np.sum(orig_data, 0) / orig_data.shape[0]
synth_col_avg = np.sum(synth_data_binomial, 0) / synth_data_binomial.shape[0]
synth_col_avg_binomial = np.sum(synth_data_binomial, 0) / synth_data_binomial.shape[0]

plt.scatter(orig_col_avg, synth_col_avg_binomial, s=25, zorder=10)
lims = [0, .5]
plt.xlim(lims)
plt.ylim(lims)
plt.plot(lims, lims, 'k-', alpha=0.5, zorder=0)
plt.xlabel("Original Data")
plt.ylabel("Synthetic Data")
plt.title("Dimension-wise Probability")
plt.show()

From the plot above we can see that we have a pretty close match on the probability distributions. This seems to result mainly from the fact that these distributions are pretty close to begin with. To verify that the distributions are the same though we can use a paired t-test to check if they are different.

From this we can see that at the 0.05 significance level. This doesn't give us the best confidence that they are different but if we were looking for proof they are different this gives us some evidence of that.

Now that we have looked at the columns we can start to look at the rows to see if that also matches.

Row Distribution¶

We will be comparing the row distributions differently from the columns. This is because each row doesn't need to have the same distribution, but we can look at the sums of the rows throughout the data to see how the counts compare to the counts in the synthetic data. We also have to scale these to be densities instead of frequencies to account for the fact that the sample sizes are different.

In [30]:
x = orig_data.sum(1)
y = synth_data_binomial.sum(1)

bins = np.linspace(0, np.max([x.max(), y.max()]), 50)

plt.hist(x, bins, alpha=0.5, label='Origonal Data', log=True, density=True)
plt.hist(y, bins, alpha=0.5, label='Synthetic Data', log=True, density=True)
plt.legend(loc='upper right')
plt.title('Comparison of Row Sum Distributions')
plt.show()

From this we can see that the densities match for a little bit but then the tail of the synthetic data is much longer than the real data. This could contribute to some of the errors that the authors were saying they saw in the data. For instance a diagnosis for a male only disease and female only disease in the same patient. There are clearly some patients who have too many diagnosis in this data.

Logistic Regression Test¶

This logistic regression test tries every column as the predicted column and tries to use the rest of the data to predict that column. The model is fitted on the origional data and then tested on the origonal data and the synthetic data. This does present problems of overfitting to the training data, but with the sparsity so small there were issues splitting the data and fitting on data without both classes. Even with the overfitting potential the data is resulting classification rate is still very good and similar across the two data sets.

Linear Regression¶

I also tried a linear regression test. In this test I created random weights from -1 to 1 for the each column of the matrix. Then I used those weights to create a prediction column for both the origonal data and the synthetic data. What I found was that the weights are slightly different but the predictions are the same. This is because, at least for the random seed being used, the origonal data is of rank 1070 instead of 1071 and for the synthtic data the rank is lower at around 880.
